# Contract Bot — Claude Code Context

## Что это за проект

Telegram-бот для автоматизации заполнения договоров туристической компании.
ИП Распопов Димитр Иосифович — групповые туры по всему миру.
Клиенты: физические и юридические лица.
Правовое поле: Российская Федерация.

## Технический стек

- **Язык**: Python 3.11+
- **Бот**: aiogram 3.x (async)
- **AI/OCR**: Claude Sonnet (anthropic SDK) — чтение паспортов с фото
- **БД**: PostgreSQL (Railway plugin)
- **ORM**: SQLAlchemy async
- **Документы**: python-docx + LibreOffice headless (DOCX → PDF)
- **Google Drive**: google-api-python-client
- **Деплой**: Railway + Dockerfile

## Пользователи и безопасность

- **Только внутренние менеджеры**: 2-3 человека
- **Авторизация**: whitelist Telegram ID в переменных окружения
- **Паспортные данные НЕ хранятся в БД** — только метаданные договора
- Данные используются только для заполнения документа, затем не сохраняются

## Нумерация договоров

- **Физлицо**: № 127 (просто номер) — свой независимый счётчик
- **Юрлицо**: № 131/кор (с суффиксом /кор) — свой независимый счётчик
- Счётчики раздельные: при первом оформлении договора каждого типа бот запрашивает стартовый номер
- Опция изменить номер вручную при генерации (единичные случаи)

## База данных — таблицы

### tour_templates
- id, name (e.g. "ОАЭ 2026"), country, city, hotel
- check_in_date, check_out_date, nights
- room_type, room_count, meal_type
- transfer (none/individual/group), insurance (bool)
- additional_conditions (text), payment_deadline
- created_at, created_by (telegram_id)

### contracts
- id, number (string, e.g. "127" или "131/кор")
- contract_date, client_name, contract_type (individual/legal)
- tour_template_id (FK), manager_telegram_id
- gdrive_file_id (nullable)
- created_at

### contract_counter
- contract_type (string, PK): "individual" или "legal"
- current_value (int) — счётчик для каждого типа отдельно
- Строки создаются при первом оформлении договора соответствующего типа

### managers
- telegram_id, name (whitelist авторизованных пользователей)

## Информация о поездке (одинакова для физ и юрлица)

- Страна и город
- Отель
- Дата заезда, дата выезда, количество ночей
- Категория номера (одноместный, двухместный, стандартный, блочный, люкс, и т.д.)
- Кол-во номеров: ½ (двухместное размещение) или 1
- Питание: BB (завтраки), AI (все включено), и др.
- Трансфер: нет / индивидуальный / групповой
- Страховка: да / нет
- Дополнительные условия (text)
- Дата полной оплаты

## Шаблоны туров

- Название: страна + год, например "ОАЭ 2026", "Саудовская Аравия 2025"
- Все менеджеры могут создавать, редактировать, удалять
- При создании нового — предложение сохранить как шаблон
- При выборе существующего — показ данных с возможностью редактировать

## Флоу: Физическое лицо

```
1. Выбор типа договора → Физлицо
2. Фото/текст РФ паспорта → Claude OCR → ФИО, серия/номер, кем выдан, дата выдачи, адрес регистрации
3. Фото загранпаспорта → Claude OCR → Фамилия/Имя латиницей, номер, дата рождения, срок действия
4. Телефон + email (текстом)
5. Показ всех данных → кнопки [Данные верные] / [Внести изменения]
   └── При исправлении: бот просит прислать корректные данные → повторная проверка
6. Заказчик по умолчанию включён в список туристов
   Бот спрашивает "Добавить ещё туристов?" (макс. 10, обычно 1-2)
7. Выбор тура: [Из сохранённых] / [Создать новый]
   └── Новый: пошаговый ввод → предложение сохранить как шаблон
8. Финансы:
   - Общая цена туристического продукта
   - Задаток (внесён в день договора)
   - Доплата = бот считает автоматически (общая - задаток)
   - Дата, до которой внести доплату
9. Дата договора: текущая дата + кнопка [Изменить]
10. Номер договора: авто + кнопка [Изменить]
    └── Если счётчик физлиц ещё не инициализирован — бот запрашивает стартовый номер
11. Генерация → отправка PDF
12. Кнопки [Отправить на Google Drive] / [Пропустить]
13. Если нужен DOCX — кнопка [Получить DOCX для редактирования]
```

## Флоу: Юридическое лицо

```
1. Выбор типа договора → Юрлицо
2. Карточка компании (фото или текст) → Claude OCR/парсинг →
   Наименование, правовая форма, ФИО директора, ИНН, КПП, ОГРН,
   юр. адрес, почт. адрес, телефон, email, банковские реквизиты
3. Проверка данных компании → [Верно] / [Исправить]
4. "Приезжающие лица" (сотрудники):
   - Для каждого: ФИО, дата рождения, номер загранпаспорта, дата выдачи, срок действия
   - Фото загранпаспорта или текстом
5. Выбор тура (аналогично физлицу)
6. Финансы: итоговая сумма (полная оплата в течение 3 банковских дней)
7. Дата договора + номер с /кор (авто)
   └── Если счётчик юрлиц ещё не инициализирован — бот запрашивает стартовый номер
8. Генерация → PDF
9. Google Drive опционально
```

## Вывод документов

- **PDF** — основной формат, отправляется когда всё верно
- **DOCX** — по запросу (кнопка), если нужно ручное редактирование
- Подпись Исполнителя: вставляется как изображение (signature.png) в нужное место
- Имя файла: "Договор № 127 Колотов Н.А.pdf" или "Договор № 131кор ТД Нарцисс.pdf"

## Google Drive

- Личный аккаунт
- Структура папок: по названию тура (страна + год)
  - Пример: папка "ОАЭ 2026" → все договора этого тура
- Авторизация через OAuth2 (сервисный аккаунт или user credentials)

## Важные решения

- Паспортные данные клиентов НЕ хранятся в БД (только в сгенерированном документе)
- Шаблоны договоров: DOCX с плейсхолдерами вида {{CONTRACT_NUMBER}}, {{CLIENT_NAME}} и т.д.
- Для конвертации DOCX→PDF используется LibreOffice headless в Docker-контейнере
- Согласие на обработку персональных данных — всегда идёт в одном файле с договором физлица, у юрлица его нет
